# image_plugin_zlhub_nano_banana

## 项目是什么

这是一个面向字字动画软件的图片中转插件，目标是直接参考现有 `nano_banana_plugin_geeknow`，实现对接 zlhub 中转平台图片大模型接口的新插件。新插件目录为 `image_plugin_zlhub_nano_banana`，与现有 geeknow 插件并存，首版以“能力 1:1 迁移”为原则。主要使用对象是团队内部成员，要求在团队使用场景下稳定出图。

## 核心价值

在不引入新功能复杂度的前提下，快速提供一个与现有 GeekNow 插件体验一致、但后端切换为 zlhub 的稳定图片生成通道。

## 需求

### 已验证

- ✓ 字字动画插件框架可承载图片中转能力并完成出图闭环 — existing
- ✓ 现有 GeekNow 插件具备主流程出图、任务日志与本地下载能力 — existing
- ✓ 团队已在当前工作流中使用该类插件形态 — existing

### 进行中

- [ ] 基于 `nano_banana_plugin_geeknow` 实现 `image_plugin_zlhub_nano_banana` 插件目录与基础结构
- [ ] 按 `docs/require/zlhub-chat-image-api.md` 对接 zlhub 图片生成接口并打通主流程出图
- [ ] 保留并对齐 GeekNow 插件的日志与任务页能力（live_log / task_log）
- [ ] 保留本地落盘下载能力与相关任务状态追踪
- [ ] 保持配置项与 UI 交互尽量与 GeekNow 插件一致
- [ ] 确保新旧插件并存可用，且不破坏原 `nano_banana_plugin_geeknow`

### 不在范围内

- 新增超出 GeekNow 现有能力的产品功能 — 首版目标是 1:1 迁移，优先稳定落地
- 改造或替换原 `nano_banana_plugin_geeknow` — 用户明确要求并存，不影响现有可用性
- 跨场景扩展（非字字动画当前插件使用场景） — 当前目标是团队内部现有场景可交付

## 背景上下文

- 当前仓库已有成熟插件 `nano_banana_plugin_geeknow`，包含 `main.py`、`ui/index.html`、`ui/task_log.html`、`ui/live_log.html`、SQLite 任务日志等完整实现。
- 已有代码库分析文档显示该插件是单模块 Python 主体 + 静态 UI 页面 + 本地 SQLite/文件系统存储的架构模式。
- zlhub 接口文档位于 `docs/require/zlhub-chat-image-api.md`，给出了 chat/completions 与图像生成示例。
- 用户在讨论中强调：首版范围为 1:1 迁移、平台切换动机为团队统一、最关键完成标准是“稳定出图”。
- 用户额外强调文档输出需为中文。

## 约束

- **范围约束**: 首版必须为 1:1 迁移，不加新功能 — 避免范围膨胀，优先保障可用性与交付确定性
- **兼容约束**: 新插件需与原插件并存 — 团队迁移期间不能影响既有流程
- **一致性约束**: 配置项与 UI 交互尽量与 GeekNow 保持一致 — 降低团队学习成本和切换成本
- **交付约束**: 必须包含主流程出图、日志任务页、本地落盘下载 — 这是用户明确指定的首版必备能力
- **文档约束**: 初始化产物需使用中文 — 用户明确要求
