<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external image URL → local filesystem | 外部 URL 内容落盘到本地目录，涉及内容完整性与路径安全 |
| plugin runtime → host result consumer | 返回路径被宿主后续流程直接消费，必须保证有效且真实存在 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | T (Tampering) | `_download_image_from_url` + save path | mitigate | 文件名固定模板且仅写入 `context['output_dir']`，不使用远端 URL 片段生成路径，避免路径注入/覆盖。 |
| T-02-06 | D (Denial of Service) | multi-image download loop | mitigate | 每次下载使用 `download_timeout`，单图失败记录后继续处理其余 URL，避免单点阻塞拖垮整批。 |
| T-02-07 | I (Information Disclosure) | error message propagation | accept | 本阶段错误回传仅含 URL/状态与简要失败原因，不含密钥；低敏感度，ASVS L1 可接受。 |
| T-02-08 | R (Repudiation) | partial success tracking | mitigate | 成功/失败逐条写 `_log_task_result`，确保每个 URL 的落盘状态可追踪。 |
</threat_model>
