<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| plugin runtime → zlhub API | 非受信任网络边界，响应体与错误消息来自外部服务 |
| host input context → request payload | `prompt/reference_images/plugin_params` 为不可信输入进入外发请求 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T (Tampering) | `send_doubao_request` payload mapping | mitigate | 固定字段白名单与固定默认值（D-01~D-04），拒绝隐式字段透传，防止请求语义被污染。 |
| T-02-02 | I (Information Disclosure) | provider error logging | mitigate | 错误信息回传时保留诊断上下文但不记录完整 Authorization/token；延续现有日志脱敏策略。 |
| T-02-03 | D (Denial of Service) | external API call | mitigate | 强制使用已有 `request_timeout`，并在非 200/结构异常时快速失败（`NO_RETRY:::`），避免长时间阻塞主链路。 |
| T-02-04 | R (Repudiation) | request/parse failure tracing | mitigate | 失败统一写入 `_log_task_result(...status='failed'...)` 并附带可诊断错误，保证审计可追溯。 |
</threat_model>
